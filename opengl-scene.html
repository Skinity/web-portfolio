Insight

This project explores core rendering techniques using OpenGL. It uses custom fragment and vertex shaders to control lighting and surface detail.

Reflections are handled through environmental maps, giving objects a more natural appearance. A swinging spotlight simulates pendulum motion through simple physics. Shadows are rendered using shadow mapping to add depth to the scene. The focus is on understanding and applying key graphics pipeline concepts.

Environmental Maps and Reflections

The scene uses dynamic cube mapping for real-time reflections. Six renders capture the environment from each cube face around the object, followed by a seventh main scene pass. The vertex shader calculates a reflection vector (r = v - 2(n â€¢ v)n) and passes it to the fragment shader, which blends reflection and base texture using mix(). This gives a responsive, subtle reflection effect.

Swinging Mechanics

The pendulum motion used to swing the ceiling lamp in the scene was done so that on each frame, the angular position (alpha) and velocity (omega) update based on Hooke's Law. The lamp rotates around a fixed point, then moves back into place.

Rigged Animation

This method uses up to 100 bone matrices, 4x4 transforms stored in a uniform array to animate models. Each Vertex carries bone IDs and weights that indicate how much each bone influences it. the shader blends these bone matrices using the weights to transform vertex positions and normals. This setup supports most animations with efficient GPU processing by applying weighted bone transforms to each vertex.
